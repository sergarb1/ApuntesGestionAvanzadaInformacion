## 📖 ¿Qué es esto?

Curso completo del **módulo 0663 — Gestión Avanzada de la Información**, perteneciente al **CFGS Asistencia a la Dirección** (Real Decreto 1583/2011, BOE 283). Orientado al perfil profesional de **Knowledge Manager**: desarrollar estrategias para captar, almacenar, organizar y difundir el capital intelectual de una organización.

Publicado como **web estática** con [Astro](https://astro.build) 7 + [Starlight](https://starlight.astro.build) 0.41 + GitHub Pages.

## 📦 Contenido del curso

| Recurso | Cantidad | Detalle |
|---------|----------|---------|
| 📚 **Unidades didácticas** | 11 | 01–11, contenido teórico-práctico (171–356 líneas c/u) |

## 🧭 Hilo conductor: Proyecto global Knowledge Manager

Todas las unidades alimentan un proyecto común: la **implantación de un Sistema de Knowledge Management** en la empresa simulada **GesDoc Consulting** (consultora de gestión documental y conocimiento). Cada unidad aporta una pieza incremental:

| Unidad | El alumno crea… |
|--------|----------------|
| **U1** 📋 | Propuesta de proyecto KM para GesDoc Consulting |
| **U2** 📊 | Planificación: cronograma, WBS, recursos, riesgos |
| **U3** 📝 | Plantillas corporativas y base de conocimiento ofimática |
| **U4** 📈 | Dashboard de indicadores e informe ejecutivo |
| **U5** 🎬 | Vídeo de presentación del proyecto KM |
| **U6** 🌐 | Espacio colaborativo en Teams/Notion/Slack |
| **U7** 🏢 | Portal web / intranet del conocimiento |
| **U8** 🗂️ | Sistema de clasificación y archivo digital |
| **U9** 🔐 | Política de custodia, flujos documentales y checklist legal |
| **U10** 🧩 | Integración de todas las piezas y memoria del proyecto |
| **U11** 🏁 | Presentación y defensa final del sistema KM completo |

> 🎯 **Objetivo:** Al final del curso, el alumno habrá diseñado e implantado un sistema KM completo para una organización real simulada, integrando conocimientos de gestión de proyectos, ofimática, comunicación audiovisual, web colaborativa, archivo digital y cumplimiento legal.

### Resultados de Aprendizaje

| RA | Descripción |
|----|-------------|
| **RA1** 📋 | Gestiona proyectos de implantación de sistemas de gestión de la información analizando sus características y aplicando metodologías de trabajo |
| **RA2** 📊 | Elabora documentos profesionales e informes empresariales utilizando aplicaciones ofimáticas y herramientas de visualización de datos |
| **RA3** 🎬 | Produce materiales de comunicación corporativa audiovisual integrando recursos multimedia |
| **RA4** 🌐 | Gestiona entornos de trabajo colaborativo y publicación web aplicando criterios de usabilidad y accesibilidad |
| **RA5** 🔐 | Organiza sistemas de archivo digital y flujos documentales garantizando la custodia legal y la seguridad de la información |

## 🚀 Desarrollo

### Requisitos

| Herramienta | Versión | Uso |
|-------------|---------|-----|
| Node.js | 24+ | Entorno de ejecución |
| npm | 10+ | Gestor de paquetes |
| Pandoc | 3+ | Generación EPUB (opcional) |
| PowerShell | 7+ | Scripts de generación (opcional) |

### Comandos

| Comando | Descripción |
|---------|-------------|
| `npm run dev` | 🚀 Servidor local (http://localhost:4321) |
| `npm run build` | 📦 Build estático en `dist/` |
| `npm run preview` | 👁️ Previsualizar build |
| `npm run pdf` | 📄 Generar PDF en `public/pdf/` |
| `npm run epub` | 📖 Generar EPUB en `public/epub/` |
| `npm run astro` | ⚡ CLI de Astro |

> **📌 Nota:** El PDF requiere el sitio accesible vía HTTP. Puedes usar `localhost:4321` modificando la URL en el comando, o desplegar en GitHub Pages primero.

## 📖 Guía de estilo del contenido

- **🎯 Tono:** Conversacional, cercano, orientado a administración y finanzas. Sin tecnicismos informáticos.
- **✏️ Lenguaje:** Empresarial, claro, práctico. "hoja de cálculo" en vez de "Excel", "documento de texto" en vez de "Word".
- **📝 Secciones especiales por unidad:**
  - `## ⭐ Ponte en la piel del Knowledge Manager` — ejercicios prácticos situacionales
  - `## ❓ Dudas frecuentes` — FAQ
  - `<aside type="note/warning/tip">` — avisos y consejos (Starlight)
  - `### Escenario:` — casos prácticos narrativos
- **📋 Formato:** Markdown compatible con Google Docs (importación directa).
- **🇪🇸 Idioma:** Castellano (raíz `/`).

## 🧪 Testing y validación

Actualmente el proyecto no cuenta con pruebas automatizadas. La validación se realiza mediante:

1. ✅ Build exitoso con `npm run build`
2. 👁️ Revisión visual del sitio generado
3. 📄 Generación correcta de PDF y EPUB

## ⚙️ CI/CD

Cada push a la rama `master` despliega automáticamente el sitio:

```yaml
# .github/workflows/deploy.yml
on: push → master | workflow_dispatch
jobs:
  build: node 24 → npm ci → npm run build → upload dist/
  deploy: GitHub Pages (origen: Actions)
```
